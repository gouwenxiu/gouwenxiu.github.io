ES6(一)

2017-05-13

ES6的介绍

  • ES6 是一个规范,不同的浏览器做了不同的实现,所以有的浏览器不一定完全支持
  • ES6的出现是为了让javascript更好.更健壮,更适应大型的应用开发
  • ES6也叫作ECMA2015

严格模式

出现时期:ES5的时候就有了

开启严格模式的代码:use strict;

作用:限制代码的编写

  • 在严格模式中函数中的this禁止指向全局对象,指向的是undefined

  • 在非严格模式下可以使用with声明,作用是简化对象用于某些对象的代码,严格模式下禁止使用with语句,报错

    严格模式和ES6

  • 严格模式是严格模式,ES6是ES6,之间没有很大的关系
  • 严格模式是代码的限制,ES6是与噶的规范,增加了相关的代码功能

  • 在ES6中的部分关键字必须在严格模式下才能使用,所以凡是ES6就使用严格模式.

    let声明变量的关键字

  • let声明变量和var声明的变量的查找机制是一样的,逐级向上查找.也就是当前作用域有就使用当前作用域的,当前作用域没有,就向上级作用域中查找

  • let是块级作用域{},不会提升成全局变量

  • 在同一个块级作用域中不允许声明同名变量.

    const声明常量的关键字

  • const是声明常量的关键字,声明常量的时候初始化是一起的;

  • 常量声明不能被第二次赋值,赋值就是被声明的对象=XXX;
    const person={}; person = "123" //报错
  • 但是可以修改挂载或者修改对象的属性
    const person={}; person.name="123"

  • 不可以同一个块级作用域声明同名变量

    let和const

  • 都是块级作用域,不会提升成全局变量

  • let是声明变量的,const是声明常量的

  • const必须将声明和初始化一起操作完毕

    模板字符串

    可以包含换行和空格,也就是原样.

    语法:包裹字符串;用反映好包裹

    特点:

  • 可以包含换行和空格
  • 可以嵌入使用变量 ${对象名.属性}

首先是ui-router的在网上的网址是(https://github.com/angular-ui/ui-router).我们可以在这里看到它的源码.

ui-router的基本用法

ui-router的基本使用方法和ng-router有很大的相同点.我们在学习的过程中要注意这些相同的点,并且区分不同的地方,下面来一一讲解:

使用步骤:

  • 引入angularjs文件
  • 引入ui-router文件
  • 创建模块,用ui-router作为依赖注入
  • 创建路由

解析:

  • 创建模块,用ui.router作为依赖注入
  • 在模块对象下,用config方法来的调用来创建路由,参数是一个数组,前面的参数是后面自调用函数的形参,这样写是防止压缩的时候参数名被压缩而导致赋值找不到名字,angularjs是根据形参名来赋值的.
  • 在这个自调用函数中有一个参数$stateProvider对象,用对象中的属性和方法来配置路由的具体参数.
  • $stateProvider对象下面有一个方法state来配置路由的的具体参数
    • url:路由的锚点
    • name:路由的名字
    • template:路由的模板 也可以是templateUrl:模板id
  • 在html中不在是用a标签的href来跳转了,而是用ui-sref命令来跳转,后面是可以是#!/锚点 ,也可以是路由的名字,一般是路由的名字,原理是会根据路由的名字来自动生成锚点的路径也就是#!/锚点,这样在维护上面比较方便.

ui-router与ngRouter不同的地方

多view方法

在ui.router中支持多view的方法,我们知道在ngRouter中这个功能是不支持的.

方法:

代码具体解析:

  • 给要填入具体的模板的html元素中设置ui-view值,这个值就是视口的名字.
  • 在state函数中,参数的对象中添加配置项views,值是一个对象,对象中内容的格式是:视口名:对象,对象中是模板的样式template字符串或者是templateUrl模板的路径或者id,也就是当前view的具体配置,还可以有controller等.
  • 没有配置具体的视口名的时候都会找没有视口名的元素中展现内容.

这样就实现了在不同的视口中显示不同的内容.

在将scope设置为一个对象的时候,表水有自己的独立作用域,但是并不会继承父级作用域中的数据,但是在想用父级作用域中的部分数据

=和@的区别

=号的时候在写指令中的属性值的时候不用写双大括号,直接写name=”name”,就可以了,这个属性也是自己自定义的.@取的时候要用上大括号包裹,这样取到的任何值都是字符串,但是有的时候传入的值不是字符串而是对象等复杂数据类型,这种时候需要用=号,然后将数据给模板来暴露数据

具体步骤:

  • 先将数据传在自定义指令所在的元素上

  • 在scope属性中取到具体的属性值
    @:取到的是字符串
    =:取到的是真实的数据类型

  • 给模板,模板在将数据暴露

    html的模板的书写位置

    • template: html字符串
    • templateUrl:模板的地址,需要开启服务器,请求数据
    • templateUrl:ngID

    transclude

    作用:在使用template和templateUrl的时候会将指令所在元素里面的内容覆盖掉,为了不覆盖掉里面的内容,可以使用transclude这个属性

    属性值:

    • true 保留指令所在元素里面的内容

    • false: 覆盖掉指令所在元素中的内容

    步骤:

    • 设置transclude属性为true
    • 告诉angularjs这里面的数据要保留并且显示,在template中书写指令<p ng-transclude></p>表示内容书写在这个p标签中

    注意:在template中,即模板中保存了要保存的数据,但是不是在页面上保存,还是删除了页面中的数据,需要再洗显示才能看到指令中保存的数据.

    replace属性

属性值:

  • true:删除指令所在的元素,只要模板,然后将指令放于模板的最外面的元素上,模板中最外面的元素上右兄弟元素的时候报错.link中element元素指向也自动替换.
  • false:默认值,不删除指令所在的元素

angular中控制器controller中的各个参数也就是服务哦名称是不能随便修改的,这个是angularjs的内部运行机制造成的.下面来详细介绍一下他的原理和附带的一些知识点,准备好一起学习了吗?

$scope 不能修改名称的原因

angularjs是使用形参名来传递参数的,找不到形参名就传递不了参数.

$scope的内部实现代码:

在angularjs中不能改变形参的名字,但是在使用压缩功能工具的时候会将形参压缩,所以在这个过程中要使用安全的方式创建构造器

angularjs的作用域

局部作用域

控制器控制的区域就是局部作用域,控制器也可以包含在控制器中,查询规则是逐级向上查找,本质上$scope就是局部作用域

全局作用域

$rootScope就是全局作用域,但是写在局部的控制器中的时候若是这个控制器没有没使用的时候就不会被创建,因此在模块对象的下面有一个函数run,来控制全局作用域

注意:
在页面加载完之后会首先执行run方法,并且这个方法值执行一次,一般作为项目的初始化,可以在$rootScope这个对象上挂载一些公用的属性和方法.
